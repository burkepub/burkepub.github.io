Mirroring This Site on Cloudflare Pages

Categories: Free Services, Other

GitHub Pages is my primary host, but I also set up Cloudflare Pages as a mirror. Here's why and how.

Why Mirror?

  • Redundancy: If one goes down, the other still works
  • Performance: Cloudflare's edge network is incredibly fast
  • Learning: It's good to know how Cloudflare Pages works

What Cloudflare Pages Offers (Free)

  • Bandwidth: Unlimited (truly unlimited - no cap)
  • Builds: 500 per month
  • Custom domains: 100 per project
  • SSL: Free automatic SSL
  • Global edge: Content served from worldwide locations

The Backup Plan

Having both GitHub Pages and Cloudflare Pages means:

  • Two live URLs for this site
  • If GitHub has issues, visitors can use Cloudflare
  • I can switch primary hosting anytime

The Trade-off

Honestly, for a simple static site like this, having both is maybe overkill. The bandwidth on GitHub Pages (100GB) is more than enough for most personal sites.

But if you want true redundancy or need unlimited bandwidth, Cloudflare Pages is excellent. And knowing how to set it up is valuable - it's become a standard tool in the web development world.

The best part? Both are completely free. You don't have to choose - you can use both.
